# TripMind AI 知识库页面原型与用户流程

## 1. 当前版本定位

知识库不是普通文件夹，也不是偏好标签堆叠页。

当前设计将知识库定义为 TripMind AI 的“可引用资料源 + 可管理记忆”：

- 资料源用于承载用户上传的攻略、预算、课程、票务截图等内容。
- AI 记忆用于沉淀用户长期偏好，但按主题折叠，避免标签无限堆叠。
- AI 问答中的“知识库”按钮是问答模式；左侧菜单中的“知识库”是资料管理页。
- 知识库里的内容后续应能被 AI 回答引用，并在回答中说明来源。

当前对应 Demo：

```text
demo/mobile-v2/index.html
```

## 3. 页面原型图

### 3.1 默认知识库首页

```text
┌────────────────────────────────┐
│ ☰             知识库       小林 │
│                                │
│ ┌────────────────────────────┐ │
│ │ 搜索  搜索资料、偏好、引用来源 │ │
│ └────────────────────────────┘ │
│                                │
│ ┌────────────────────────────┐ │
│ │ 上传资料                 + │ │
│ │ 攻略、预算、课程计划、票务截图 │ │
│ │ 都可以作为 AI 回答依据       │ │
│ └────────────────────────────┘ │
│                                │
│ ┌────────────────────────────┐ │
│ │ 资料源              4 个可引用 │ │
│ │ ┌────┐ 云南游学攻略.pdf      │ │
│ │ │PDF │ 攻略 · 12 条知识      │ │
│ │ └────┘ 可用于行程规划        │ │
│ │ ┌────┐ 个人预算计划.xlsx     │ │
│ │ │ 表 │ 预算 · 识别预算上限   │ │
│ │ └────┘                      │ │
│ └────────────────────────────┘ │
│                                │
│ ┌────────────────────────────┐ │
│ │ AI 记忆              按主题折叠 │ │
│ │ ▾ 预算与时间             2 条 │ │
│ │ ▸ 饮食与身体限制         2 条 │ │
│ │ ▸ 目的地偏好             2 条 │ │
│ │ ▸ 风险偏好               1 条 │ │
```

设计意图：

- 搜索在最上方，承接资料多了之后的查找需求。
- 上传入口保持显眼，但不占据整个页面。
- 资料源优先展示“可被 AI 引用的文件/内容”。
- AI 记忆按主题折叠，不直接展示大量标签。

### 3.2 搜索态

```text
┌────────────────────────────────┐
│ 搜索  云南                       │
├────────────────────────────────┤
│ 资料源                          │
│ [PDF] 云南游学攻略.pdf           │
│ [记]  小林的行程记忆             │
│                                │
│ AI 记忆                         │
│ 目的地偏好 > 希望兼顾自然风景... │
└────────────────────────────────┘
```

规则：

- 搜索范围包括资料标题、资料类型、AI 记忆内容和引用说明。
- 搜索结果后续可以按“资料源 / AI 记忆 / 引用”分组。
- Demo 当前只展示搜索框，不实现真实过滤。

### 3.3 上传资料态

```text
┌────────────────────────────────┐
│ 上传资料                    +  │
│                                │
│ ┌────────┐ ┌────────┐ ┌────────┐ │
│ │  拍照   │ │  相册   │ │ 本地文件 │ │
│ └────────┘ └────────┘ └────────┘ │
│                                │
│ 上传后进入解析队列：             │
│ - 识别资料类型                  │
│ - 提取关键知识                  │
│ - 生成可引用来源                │
│ - 可能更新 AI 记忆              │
└────────────────────────────────┘
```

规则：

- 上传入口可以复用 AI 问答页的附件能力。
- 资料上传后先进入解析流程，不直接变成散乱标签。
- 解析结果应进入“资料源”，同时提取出的长期偏好进入“AI 记忆”。

规则：

- 每个资料源都要显示类型、标题、摘要和可用于什么场景。
- 后续点击资料源可进入详情页，查看提取知识和引用片段。
- AI 回答应能引用资料源标题或片段，提高可信度。

### 3.5 AI 记忆折叠态

```text
┌────────────────────────────────┐
│ AI 记忆                  按主题折叠 │
│                                │
│ ▾ 预算与时间             2 条   │
│   - 预算上限 15000 元           │
│   - 每天保留 2 小时学习时间      │
│                                │
│ ▸ 饮食与身体限制         2 条   │
│ ▸ 目的地偏好             2 条   │
│ ▸ 风险偏好               1 条   │
└────────────────────────────────┘
```

规则：

- 默认只展开最常用或最近更新的一组。
- 其他组折叠展示数量，降低视觉负担。
- 每条记忆后续应支持来源追溯、编辑、删除或停用。
- 记忆不是所有关键词的合集，只保存能影响后续决策的长期信息。

## 4. 用户流程

### 4.1 从 AI 问答进入知识库管理

```text
用户在 AI 问答页
  ↓
点击左上角菜单
  ↓
打开左侧菜单
  ↓
点击“知识库”
  ↓
进入知识库管理页
  ↓
查看资料源和 AI 记忆
```

### 4.2 上传并整理资料流程

```text
用户进入知识库
  ↓
点击“上传资料”
  ↓
选择拍照 / 相册 / 本地文件
  ↓
系统识别资料类型
  ↓
提取关键知识
  ↓
生成资料源摘要
  ↓
必要时更新 AI 记忆分组
```

### 4.3 使用资料源辅助问答流程

```text
用户在 AI 问答页点亮“知识库”模式
  ↓
输入问题
  ↓
AI 检索知识库资料源和 AI 记忆
  ↓
AI 生成回答
  ↓
回答中标注引用依据
```

### 4.4 管理 AI 记忆流程

```text
用户进入知识库
  ↓
查看 AI 记忆
  ↓
展开某个记忆分组
  ↓
查看具体记忆条目
  ↓
后续可编辑 / 删除 / 停用 / 查看来源
```

### 4.5 搜索资料流程

```text
用户进入知识库
  ↓
在搜索框输入关键词
  ↓
系统同时搜索资料源和 AI 记忆
  ↓
展示匹配结果
  ↓
用户点击结果查看详情或回到 AI 问答继续提问
```

## 5. 当前设计共识

- 知识库不是标签墙，不能把所有偏好直接铺出来。
- 知识库首页优先展示“资料源”和“AI 记忆分组”。
- AI 记忆应按主题折叠，并支持后续管理。
- 资料源应具备可引用性，后续 AI 回答要能说明依据来自哪里。
- AI 问答页的“知识库”是问答模式，左侧菜单的“知识库”是管理页面。
