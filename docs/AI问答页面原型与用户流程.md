# TripMind AI 问答页面原型与用户流程

## 1. 当前版本定位

当前 AI 问答页是 TripMind AI 的主入口。

页面不再采用底部多模块导航，而是采用“聊天助手首页 + 模式按钮 + 常驻输入框 + 左侧菜单”的结构：

- 用户默认从 AI 问答开始表达需求。
- “行程规划 / Plan B / 知识库”不是页面跳转，而是 AI 分析模式。
- 知识库管理入口放入左侧菜单。
- 输入框负责承接文本、语音、附件三种输入方式。

当前对应 Demo：

```text
demo/mobile-v2/index.html
```

## 3. 页面原型图

### 3.1 默认文本输入态

```text
┌────────────────────────────────┐
│ ☰          TripMind AI     小林 │
│                                │
│                                │
│   ◲  我是 TripMind AI          │
│      陪你规划游学、对比方案、    │
│      处理途中变化。             │
│                                │
│   ┌────────────────────────┐   │
│   │ 新疆 15 天自驾和云南...  │   │
│   └────────────────────────┘   │
│   ┌──────────────────┐         │
│   │ 预算 1.5 万...    │         │
│   └──────────────────┘         │
│   ┌──────────────────────┐     │
│   │ 遇到天气变化...       │     │
│   └──────────────────────┘     │
│                                │
│ [行程规划] [Plan B] [知识库]    │
│ ┌────────────────────────────┐ │
│ │ 语音  输入你的旅行问题... + 发送 │ │
│ └────────────────────────────┘ │
└────────────────────────────────┘
```

规则：

- 输入框初始为一行高度。
- 用户输入超过一行后，输入框向上扩展。
- 空文本时，“发送”按钮禁用并置灰。
- 有文本后，“发送”按钮变为可点击。

### 3.2 模式点亮态

```text
┌────────────────────────────────┐
│ [行程规划*] [Plan B] [知识库]   │
│ ┌────────────────────────────┐ │
│ │ 语音  新疆和云南怎么选？  + 发送 │ │
│ └────────────────────────────┘ │
└────────────────────────────────┘
```

规则：

- 点击模式按钮后，仅点亮按钮。
- 不额外出现“已启用 xxx 模式”的文案。
- 模式本质是 AI 输出策略，不是页面跳转。
- 用户输入中出现明显关键词时，后续可通过 prompt 自动点亮模式。

模式说明：

| 模式 | 触发方式 | AI 输出倾向 |
| --- | --- | --- |
| 行程规划 | 点击“行程规划”，或用户输入行程/路线/规划/游学等关键词 | 输出结构化行程建议、日程节奏、注意事项 |
| Plan B | 点击“Plan B”，或用户输入天气/突发/改签/退票等关键词 | 输出风险识别、替代方案、后续操作建议 |
| 知识库 | 点击“知识库”，或用户输入攻略/资料/文件/笔记等关键词 | 优先结合已上传资料和记忆回答 |

### 3.3 语音输入态

```text
┌────────────────────────────────┐
│ [行程规划] [Plan B] [知识库]    │
│ ┌────────────────────────────┐ │
│ │ 键盘        按住说话       × │ │
│ └────────────────────────────┘ │
└────────────────────────────────┘
```

规则：

- 点击“语音”后进入语音输入态。
- 语音态覆盖原文本输入框。
- 点击“键盘”或“×”返回文本输入态。
- Demo 阶段仅展示交互状态，不接入真实录音能力。

### 3.4 附件面板态

```text
┌────────────────────────────────┐
│ [行程规划] [Plan B] [知识库]    │
│ ┌────────────────────────────┐ │
│ │ 语音  输入你的旅行问题... + 发送 │ │
│ └────────────────────────────┘ │
│ ┌────────┐ ┌────────┐ ┌────────┐ │
│ │  拍照   │ │  相册   │ │ 本地文件 │ │
│ └────────┘ └────────┘ └────────┘ │
└────────────────────────────────┘
```

规则：

- 点击“+”展开附件面板。
- 再次点击“+”可收起。
- 附件入口包含：拍照、相册、本地文件。
- Demo 阶段仅模拟选择动作，不接入真实上传。

### 3.5 左侧菜单态

```text
┌────────────────────────────────┐
│┌──────────────────────┐        │
││ TripMind AI        × │        │
││                      │        │
││ [AI 问答]            │        │
││ [知识库]             │        │
│└──────────────────────┘        │
└────────────────────────────────┘
```

规则：

- 点击左上角菜单打开左侧菜单。
- 知识库作为资料管理页入口放在左侧菜单。
- 点击“知识库”进入知识库界面，聊天输入区隐藏。

## 4. 用户流程

### 4.1 默认问答流程

```text
用户进入 App
  ↓
看到 AI 问答首页
  ↓
输入旅行问题
  ↓
发送按钮从禁用变为可用
  ↓
点击发送
  ↓
AI 按通用问答逻辑理解需求并生成回复
```

### 4.2 模式问答流程

```text
用户进入 AI 问答页
  ↓
点击“行程规划 / Plan B / 知识库”
  ↓
对应模式按钮点亮
  ↓
用户输入问题
  ↓
点击发送
  ↓
AI 按当前模式生成更结构化的分析结果
```

### 4.3 自动识别模式流程

```text
用户未手动选择模式
  ↓
用户输入：“新疆和云南 15 天游学怎么规划？”
  ↓
系统识别“游学 / 规划”等关键词
  ↓
自动点亮“行程规划”
  ↓
AI 按行程规划模式输出
```

后续需要通过真实 prompt 和模型效果调试自动识别规则，Demo 当前只做轻量关键词模拟。

### 4.4 语音输入流程

```text
用户点击“语音”
  ↓
文本输入框切换为语音输入界面
  ↓
用户按住说话
  ↓
语音转文字
  ↓
文字填入输入框
  ↓
用户确认并发送
```

Demo 当前只展示“语音输入界面”，不实现真实语音识别。

### 4.5 附件上传流程

```text
用户点击“+”
  ↓
出现“拍照 / 相册 / 本地文件”
  ↓
用户选择资料来源
  ↓
资料进入知识库解析队列
  ↓
AI 后续问答可引用资料内容
```

Demo 当前只展示入口，不实现真实文件读取。

### 4.6 知识库管理流程

```text
用户点击左上角菜单
  ↓
打开左侧菜单
  ↓
点击“知识库”
  ↓
进入知识库页面
  ↓
查看上传入口、AI 已记住的偏好、已整理资料
```

## 5. 当前设计共识

- AI 问答是主入口，用户不需要先选择复杂模块。
- 行程规划、Plan B、知识库是“模式”，不是跳转链接。
- 知识库作为资料管理页面，放在左侧菜单中。
- 空输入不允许发送，避免无效操作。
- 输入方式先覆盖文本、语音、附件三类，后续再细化真实能力。
