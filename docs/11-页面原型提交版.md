## 1. 文档说明

本文档作为作业交付物中的“页面原型”部分，包含：

- 核心页面线框图。
- AI 与用户互动方式。
- 输入方式、反馈机制和结果呈现。
- Demo 访问方式。

当前页面原型为低保真移动端 App 原型，重点展示核心交互流程和 AI 输出结构，不追求完整商业化视觉稿。

## 2. 原型范围

本原型覆盖以下页面和状态：

| 页面 / 状态 | 说明 |
| --- | --- |
| 启动首页 | 建立产品第一印象，引导用户点击“出发” |
| AI 问答首页 | 统一 AI 入口，支持自然语言输入和模式选择 |
| 行程规划结果 | 展示推荐结论、横向滑动多维对比、备选方案、来源与不确定项 |
| 知识库管理页 | 展示资料源、AI 记忆分组和记忆确认 |
| Plan B 结果 | 展示突发影响判断、替代方案、成本变化、执行确认清单 |
| 左侧菜单 | 在 AI 问答和知识库之间切换 |
| 对话历史 | 后续版本补充，用于查看和继续过往对话 |

## 4. 页面 2：AI 问答主入口

### 4.1 线框图

```text
┌────────────────────────────────┐
│ 9:41                    5G 100% │
├────────────────────────────────┤
│  ☰          TripMind AI    小林 │

│              陪你规划游学、对比 │

│                                │
│  [新疆 15 天自驾和云南 15 天游学怎么选？] │

│                                │
├────────────────────────────────┤
│ [行程规划] [Plan B] [知识库]    │
│ ┌────────────────────────────┐ │
│ │ 语音  输入你的旅行问题... + 发送 │ │
│ └────────────────────────────┘ │

└────────────────────────────────┘
```

### 4.2 交互说明

- AI 问答是所有 AI 能力的统一入口。
- 用户可以直接输入自然语言问题。
- 用户也可以手动点亮输入框上方的模式按钮：
  - 行程规划
  - Plan B
  - 知识库
- 如果用户不手动选择模式，系统根据关键词和上下文自动识别。
- 输入框回车发送，`Shift + Enter` 换行。
- 附件入口用于模拟上传攻略、表格、截图等资料。

- 来源与不确定项默认折叠，用户需要时可展开。

- 成本和时间变化使用范围估算，不假装精确。
- 执行前确认清单用于承接高风险操作。
- AI 不会自动取消预约、改签订单或通知同行人。

## 9. 后续版本：对话历史

对话历史页会在 MVP 后续阶段补充，主要用于保存和管理用户过往的行程规划、Plan B 和知识库问答。

它不是当前 MVP 跑通的核心功能。当前 MVP 更关注用户是否能通过 AI 问答入口完成一次完整的“提出问题 → 获取结构化结果 → 查看来源和不确定项 → 做出确认”的主链路。

后续版本中的对话历史建议包含：

- 历史会话列表，按最近时间排序。
- 会话标题自动生成，例如“新疆 vs 云南 15 天游学对比”。
- 支持搜索目的地、时间、资料源和关键词。
- 支持从历史会话继续追问，减少重复输入。
- 支持删除、重命名和置顶重要会话。

## 12. 原型设计总结

本原型重点验证三件事：

- 用户是否能通过一个 AI 问答入口完成规划、资料管理和 Plan B。
- AI 长回复是否能通过卡片化结构降低阅读压力。
- AI 的来源、置信度、不确定项和确认清单是否能提升可信度与控制感。

对话历史会提升长期使用体验，但不是本次 MVP 的主链路，因此放入后续版本规划。
