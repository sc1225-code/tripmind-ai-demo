# TripMind AI 页面原型线框说明

## 1. 原型目标

本原型用于展示 TripMind AI 的 MVP 核心体验：用户通过自然语言提出复杂行程决策问题，上传旅行资料，AI 自动整理信息并生成可编辑记忆卡片，随后输出多方案对比报告、推荐行程概要，并在突发天气事件下生成 Plan B 与执行前确认清单。

原型形态：

- 设备端：Web 桌面端
- 页面类型：单页面流程式 Demo
- 视觉方向：AI 工作台
- 交互方式：CUI + GUI 混合
- 资料导入：上传按钮 + 示例文件状态
- Plan B 触发：流程末段展示，同时通过页面右上角突发事件提醒增强主动感

## 2. 页面整体结构

页面采用 AI 决策工作台布局。

```text
┌────────────────────────────────────────────────────────────────────┐
│ Top Bar: TripMind AI / 当前任务 / 突发事件提醒 / 用户状态             │
├───────────────┬──────────────────────────────────────┬─────────────┤
│ 左侧流程导航   │ 中央主工作区                           │ 右侧状态栏   │
│               │                                      │             │
│ 1 输入需求     │ Step 1 复杂需求输入                    │ 当前记忆     │
│ 2 上传资料     │ Step 2 资料上传与解析                  │ 可信状态     │
│ 3 确认记忆     │ Step 3 记忆卡片确认                    │ 数据来源     │
│ 4 方案对比     │ Step 4 多方案对比报告                  │ 待确认项     │
│ 5 行程确认     │ Step 5 推荐行程概要                    │             │
│ 6 Plan B      │ Step 6 突发事件与 Plan B               │             │
└───────────────┴──────────────────────────────────────┴─────────────┘
```

### 2.1 顶部栏

顶部栏用于说明当前产品和任务状态。

包含内容：

- 产品名称：TripMind AI
- 当前任务：新疆 15 天自驾 vs 云南 15 天游学
- 状态标签：正在生成决策报告 / 已生成推荐方案 / 检测到天气变化
- 突发事件提醒入口：大理 Day 5 暴雨风险
- 用户状态：小林，15 天旅行计划

### 2.2 左侧流程导航

左侧导航用于体现单页面流程进度。

流程节点：

1. 输入需求
2. 上传资料
3. 确认记忆
4. 方案对比
5. 行程确认
6. Plan B

每个节点展示状态：

- 未开始
- 处理中
- 已完成
- 需要确认

### 2.3 中央主工作区

中央区域是主要交互和结果展示区。用户按流程向下完成任务，每个区块既可以被滚动查看，也可以通过左侧导航快速定位。

### 2.4 右侧状态栏

右侧状态栏常驻展示当前 AI 使用的核心上下文。

包含内容：

- 当前用户记忆
- 已使用资料来源
- 结论置信度
- 不确定项
- 待用户确认事项

该区域用于强化 TripMind AI 的可信机制，让用户知道 AI 的判断依据和边界。

## 3. Step 1 复杂需求输入

### 3.1 设计目标

让用户用自然语言开始复杂行程决策，而不是填写传统 OTA 表单。

### 3.2 线框结构

```text
┌──────────────────────────────────────────────┐
│ 复杂行程决策                                 │
│ 让 AI 帮你把纠结的旅行方案变成可解释决策。      │
│                                              │
│ ┌──────────────────────────────────────────┐ │
│ │ 我在新疆 15 天自驾和云南 15 天游学之间纠结， │ │
│ │ 预算 1.5 万，不吃牛羊肉，容易晕车，希望每天  │ │
│ │ 保留 2 小时学习时间，帮我分析哪个更适合。    │ │
│ └──────────────────────────────────────────┘ │
│                                              │
│ [开始分析] [使用示例需求]                     │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│ AI 已识别的任务信息                           │
│ 方案 A：新疆 15 天自驾                         │
│ 方案 B：云南 15 天游学                         │
│ 预算：15000 元                                │
│ 饮食限制：不吃牛羊肉                           │
│ 体力限制：容易晕车                             │
│ 学习目标：每天 2 小时                          │
│ 待补充：出发城市、具体日期、同行人数             │
└──────────────────────────────────────────────┘
```

### 3.3 关键交互

- 用户可以直接编辑自然语言输入。
- 点击“开始分析”后，AI 自动抽取目的地、时间、预算、偏好和约束。
- 未识别的信息进入“待补充”区域，而不是打断用户流程。

## 4. Step 2 资料上传与解析

### 4.1 设计目标

展示 AI 如何解决信息碎片化问题，将攻略、计划和预算资料整理成可用于决策的知识条目。

### 4.2 线框结构

```text
┌──────────────────────────────────────────────┐
│ 上传旅行资料                                 │
│ 支持攻略、预算表、课程计划、政策说明等。        │
│                                              │
│ ┌──────────────────────────────────────────┐ │
│ │ 拖拽文件到这里，或点击上传                 │ │
│ │ 支持 PDF / DOCX / XLSX / TXT              │ │
│ └──────────────────────────────────────────┘ │
│                                              │
│ 示例文件状态                                  │
│ ✓ 新疆自驾攻略.pdf        已解析               │
│ ✓ 云南游学攻略.pdf        已解析               │
│ ✓ 个人预算计划.xlsx       已解析               │
│ ✓ 学习计划.docx           已解析               │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│ AI 解析出的知识条目                           │
│ 类型      内容                         来源    │
│ 预算      新疆租车与油费占比较高          新疆攻略 │
│ 风险      新疆长距离自驾对晕车用户不友好   新疆攻略 │
│ 学习      云南更容易保留固定学习时间      学习计划 │
│ 天气      云南雨季需准备室内 Plan B       云南攻略 │
└──────────────────────────────────────────────┘
```

### 4.3 关键交互

- 上传按钮在 MVP 中可接受文件，但 Demo 可使用预置示例文件模拟解析结果。
- 每条知识条目展示来源标签。
- 用户可以展开某条知识，查看原始资料片段。

## 5. Step 3 记忆卡片确认

### 5.1 设计目标

展示 TripMind AI 的长期记忆能力，同时体现用户对记忆的控制权。

### 5.2 线框结构

```text
┌──────────────────────────────────────────────┐
│ AI 识别到以下会影响行程决策的记忆              │
│                                              │
│ ┌──────────────┐ ┌──────────────┐             │
│ │ 预算上限      │ │ 饮食限制      │             │
│ │ 15000 元      │ │ 不吃牛羊肉    │             │
│ │ 本次行程      │ │ 长期偏好      │             │
│ │ [编辑] [删除] │ │ [编辑] [删除] │             │
│ └──────────────┘ └──────────────┘             │
│                                              │
│ ┌──────────────┐ ┌──────────────┐             │
│ │ 体力限制      │ │ 学习目标      │             │
│ │ 容易晕车      │ │ 每天 2 小时   │             │
│ │ 长期偏好      │ │ 本次行程      │             │
│ │ [编辑] [删除] │ │ [编辑] [删除] │             │
│ └──────────────┘ └──────────────┘             │
│                                              │
```

### 5.3 关键交互

- 记忆卡片支持编辑和删除。
- 用户可选择记忆类型：本次行程 / 长期偏好。
- AI 自动提取记忆，但必须经用户确认后用于后续决策。

## 6. Step 4 多方案对比报告

### 6.1 设计目标

展示 AI 的核心决策能力，让用户看到推荐结论、对比依据、来源和不确定项。

### 6.2 线框结构

```text
│ 推荐结论                                     │

│ 自驾体验更强，但长距离移动对晕车用户不友好。     │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│ 多维度对比                                   │
│ 维度          新疆 15 天自驾      云南 15 天游学 │
│ 预算匹配       中，租车油费高       高，更可控    │
│ 体力压力       高，移动距离长       中，节奏灵活  │

│ 饮食适配       中，限制较多         高，选择更多  │

┌──────────────────────────────────────────────┐
│ 可信说明                                     │

│ 路线距离估算                                  │

│ 不确定项：实时机票价格、住宿波动、课程名额、景点 │
```

### 6.3 关键交互

- 用户可以展开每个维度查看依据。
- 用户可以调整权重，例如更看重预算、学习或体验。
- MVP 中优先展示固定维度，不做复杂权重配置。

## 7. Step 5 推荐行程概要

### 7.1 设计目标

将推荐结论转化为可执行的概要行程，完成从决策到计划的过渡。

### 7.2 线框结构

```text
┌──────────────────────────────────────────────┐
│ 云南 15 天游学概要行程                         │
│                                              │
│ Day 1-2    昆明抵达与适应                      │
│ Day 3-6    大理游学与轻户外                    │
│ Day 7-10   丽江文化体验与学习时间               │
│ Day 11-13  香格里拉轻路线或备用路线             │
│ Day 14     昆明复盘与返程准备                  │
│ Day 15     返程                               │
│                                              │
│ [确认使用该方案] [调整节奏] [降低预算]           │
└──────────────────────────────────────────────┘
```

### 7.3 关键交互

- 用户可以确认使用推荐方案。
- MVP 中保留“调整节奏”和“降低预算”按钮，但可用模拟反馈。
- 行程概要展示学习时间块和风险提醒，避免只是普通旅游路线。

## 8. Step 6 突发事件与 Plan B

### 8.1 呈现方式建议

Plan B 作为流程最后一个区块展示，同时在顶部栏右侧出现突发事件提醒。

推荐原因：

- 保持单页面流程清晰。
- 让评审能完整看到“规划到动态调整”的闭环。
- 右上角提醒能体现 AI 的主动监控感。
- 实现难度低于弹窗或复杂实时通知。

### 8.2 线框结构

```text
┌──────────────────────────────────────────────┐
│ 突发事件提醒                                  │
│ Day 5 大理出现持续暴雨，原计划洱海骑行不建议进行。│
│                                              │
│ 影响范围：                                    │
│ - 原计划：洱海骑行 + 户外拍摄                  │
│ - 影响：安全风险升高，体验质量下降              │
│ - 影响时间：Day 5 下午                         │
│ - 预算影响：部分费用可取消或顺延                │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│ TripMind AI 推荐 Plan B                       │
│ 推荐方案：                                    │
│ 上午：保留原课程/学习安排                      │
│ 下午：非遗体验馆 + 室内咖啡馆学习               │
│ 晚上：根据天气决定是否短途散步                  │
│                                              │
│ 备选方案：                                    │
│ 1. 云南省博物馆 + 课程复盘                     │
│ 2. 民宿内学习半日 + 晚间轻餐                   │
│                                              │
│ 推荐理由：符合轻节奏、学习目标和预算约束，且减少 │
│ 雨天户外安全风险。                             │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│ 执行前确认清单                                │
│ □ 取消洱海骑行预约                            │
│ □ 新增非遗体验预约                            │
│ □ 调整下午打车时间                            │
│ □ 通知同行人行程变化                          │
│ □ 更新 Day 5 预算                             │
│                                              │
│ [确认生成调整方案] [暂不处理]                  │
└──────────────────────────────────────────────┘
```

### 8.3 关键交互

- 用户可以勾选需要执行的事项。
- MVP 不做真实取消、预订、打车或通知。
- 涉及订单和金钱的操作必须停留在用户确认前。

## 10. 原型页面优先级

MVP 原型必须重点打磨：

- 复杂需求输入
- 资料上传与解析
- 记忆卡片确认
- 多方案对比报告
- Plan B 与执行前确认清单

可简化处理：

- 完整 15 天每日详细行程
- 权重自定义
- 真实文件解析
- 真实 API 调用
- 真实订单执行

## 11. 原型评审时的讲述顺序

建议演示时按以下顺序讲述：

1. 用户不是来“查机票”，而是在做复杂旅行决策。
2. 用户用自然语言提出新疆 vs 云南的问题。
3. AI 引导上传资料，并将攻略与计划结构化。
4. AI 提取记忆卡片，但需要用户确认后使用。
5. AI 输出可解释的多方案对比报告。
6. 用户接受云南游学推荐，进入概要行程。
7. 途中触发大理暴雨事件，AI 生成 Plan B。
8. AI 输出执行前确认清单，但不越权自动下单或改签。
